**Компромисс:** Эти гайдлайны смещены в сторону осторожности, а не скорости.
Для тривиальных задач (опечатка, очевидный однострочник) — используй здравый смысл.

- Формулируй предположения явно. Если не уверен — спрашивай.
- Если есть несколько интерпретаций — представляй все, не выбирай молча.
- Если есть более простой подход — скажи об этом. Возражай, когда нужно.

- Если не понимаешь, как работает конкретный MT5-вызов — спроси, не угадывай.
- Расчёт лота чувствителен к нулевым значениям (tick_size, tick_value, volume_step).

**Трогай только то, что должен. Убирай только за собой.**

При редактировании существующего кода:
- Не «улучшай» соседний код, комментарии или форматирование.
- Не рефакторь то, что не сломано.
- Следуй существующему стилю, даже если сделал бы иначе.
- Если заметил мёртвый код — упомяни, не удаляй.

Когда твои изменения создают сирот:
- Удаляй импорты/переменные/функции, которые ТВОИ изменения сделали неиспользуемыми.
- Не удаляй ранее существовавший мёртвый код, если не попросили.

Тест: каждая изменённая строка должна напрямую отслеживаться до запроса пользователя.

Стиль кода проекта:
- Язык: Python 3.x
- GUI: tkinter
- Кодировка: UTF-8
- Комментарии и docstrings на русском
- GUI-строки на русском
- Типизация: Optional, Dict, List из typing (не from __future__ import annotations)
- Импорты: try/except для MetaTrader5 и psutil (они могут быть не установлены)
- Логирование: через callback-функции, не через logging module напрямую
- Разделители секций: `# ───────────────...`

## Специфика проекта: MT5 Copy Trader

### Архитектура

```
copytrade/
├── gui.py        — GUI на tkinter, точка входа
├── copier.py     — логика копирования (фоновый поток)
├── config.json   — настройки (автосоздание)
├── state.json    — маппинг тикетов (автосоздание)
├── requirements.txt
├── build.bat     — PyInstaller сборка
└── README.md
```

### Ключевые правила

1. **GUI на русском** — все надписи, кнопки, сообщения
2. **Лог с эмодзи** — ✅ успех, ⚠️ предупреждение, ❌ ошибка
3. **Статус с эмодзи** — 🟢 подключён (#аккаунт), 🔴 не подключён
4. **config.json** сохраняется при нажатии Старт, загружается при старте приложения
5. **Фоновый поток** останавливается через threading.Event
6. **При ошибке подключения к мастеру** — логировать и продолжать попытки, не падать
7. **При ошибке order_send** — логировать retcode и описание
8. **Комментарий к сделкам слейва**: `CT_{master_ticket}`
9. **Отложенные ордера**: TRADE_ACTION_PENDING
10. **Разные символы** (XAUUSD → GOLD): SL/TP не копировать (0)
11. **Одинаковые символы**: SL/TP копировать как есть

### Расчёт лота (PositionSizer)

```
SL_distance = abs(open_price - sl_price)
SL_ticks    = SL_distance / trade_tick_size
risk_amount = balance * risk_percent / 100  (или фиксированная сумма)
lot         = risk_amount / (SL_ticks * trade_tick_value)
lot         = round до volume_step
lot         = clamp [volume_min, volume_max]
```

Если SL не задан → default_lot из настроек слейва.

### State format

```json
{
  "positions": {"111111": {"acc1": 222222}},
  "orders":    {"444444": {"acc1": 555555}}
}
```

Ключи — строки (str(master_ticket)). Значения — dict {slave_id: slave_ticket (int)}.
